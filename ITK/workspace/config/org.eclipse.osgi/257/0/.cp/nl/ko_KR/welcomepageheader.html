제품 시각화...


BMIDE 시작! BMIDE는 데이터 모델 및 동작을 보고 확장하는 데 사용되며, 유효성 검증을 위한 테스트 Teamcenter 환경에 배치하고, 사용자의 확장으로 이동해서 라이브 생산 환경에서 확장을 편집하는 데 사용됩니다. BMIDE는 이러한 작업을 완료하도록 돕습니다.

시작하려면 프로젝트 생성이 필요합니다. 프로젝트는 BMIDE가 완료하려는 작업 유형을 이해하도록 돕고 모든 확장을 위한 컨테이너로 작동합니다. 아래의 다음 프로젝트 유형 중 하나를 클릭하여 사용된 것을 확인합니다. 생성하려는 프로젝트의 유형을 정의한 다음, 생성 버튼을 클릭하여 시작합니다.

새 BMIDE 템플릿 생성


Teamcenter 확장이 처음이라면 먼저 템플릿 프로젝트를 생성해야 합니다. 템플릿은 BMIDE 내에서 생성한 모든 비즈니스 개체, 속성 및 동작에 대해 컨테이너로 작동합니다. 테스트용 Teamcenter 환경에 이 템플릿을 배치할 수 있습니다. 유효성 검증 후 사용자가 사용할 수 있도록 템플릿을 생산 현장에 설치할 수 있습니다.

기존 BMIDE 템플릿에 대한 라이브 업데이트 수행


라이브 실행 생산 현장의 동작을 편집하려는 경우. 예를 들어 LOV 값, 상태 또는 ID 컨텍스트를 추가하려는 경우. BMIDE는 Teamcenter 시스템을 분해하지 않고 라이브 업데이트를 수행하기 위해 프로젝트의 이 유형을 사용합니다. 프로젝트의 이 유형을 사용하기 전에 생산 현장(위 참조)에 템플릿 프로젝트를 생성하고 설치해야 합니다.

이전 Teamcenter 릴리스에서 BMIDE 템플릿 업그레이드


사용자의 Teamcenter 설치를 새 Teamcenter 버전으로 업그레이드 진행 중이면 먼저 BMIDE 템플릿을 업그레이드해야 합니다. 템플릿의 이전 버전이 Teamcenter의 새 버전과 호환되지 않아서 템플릿 업그레이드가 필요합니다. BMIDE 템플릿을 업그레이드하려면 먼저 BMIDE 클라이언트의 새 버전을 설치한 다음, “템플릿 업그레이드” 아이콘을 클릭하고 마법사를 사용하여 현재 BMIDE 템플릿을 새 릴리스로 업그레이드합니다. 업그레이드 단계의 완전한 설정에 대해서는 “업그레이드 가이드”를 참조하십시오.
