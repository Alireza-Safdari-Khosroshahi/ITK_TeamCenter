Update of Batch LOVs

Batch LOVs are LOVs the values of which are updated in the Database by a command line utility. The LOVs are still defined and attached to the properties of Business Objects in the BMIDE project but the values are managed external to the BMIDE.

Following is the complete process for working with Batch LOV

Creation of a Batch LOV

In the LOV Wizard choose the LOV Value Management option as "Supply values directly to Teamcenter database using "bmide_manage_batch_lovs" command line utility"

Since the template should not contain LOV values for Batch LOV, the UI to add LOV values will be disabled. After specifying other mandatory fields click finish to create the LOV.

Once you have created a Batch LOV, BMIDE automatically creates sample XML files which you may use to update the LOV values of this LOV by a command line utility.

The sample files are created in the folder <BMIDEProject>/output/Samples . Please review the XML files.

Deploy

Once you have the Batch LOV defined and attached to the properties, you can deploy the template to the Database.

Prepare XML files for the Batch LOV

Using the sample files created in the BMIDE, create input XML files that will contain the LOV values and localization for the LOV values and description..

You can also refer to samples and the readme available in $TC_ROOT/bmide/samples directory.

Please note the XML files created should conform to the XSD published by BMIDE for the Batch LOV. BatchBusinessData.xsd and BatchBusinessDataLocalization.xsd will be available in the $TC_DATA directory

Update LOV values using utility

Once you have Batch LOVs deployed to Database, use the command line utility bmide_manage_batch_lovs with the input XML files that you have created.

The command is: bmide_manage_batch_lovs –option=update –file=<XML file path> -log=>log file path>
